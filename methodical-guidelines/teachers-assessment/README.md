# Рекомендации преподавателям по оцениванию

## Общие принципы

1. Оценивается **наблюдаемый результат**, а не старательность: каждый критерий рубрики сформулирован через проверяемое действие.
2. Балл выставляется **по рубрике КИМ**, а не по общему впечатлению; при промежуточных значениях выбирается ближайшая ступень.
3. Все баллы согласованы с [итоговой таблицей](../../README.md#4-итоговая-оценка): суммы по рубрикам не должны расходиться с максимумами КИМ.
4. Результаты доводятся до обучающихся в течение недели после сдачи с краткой письменной обратной связью по каждому критерию.

## Процедура оценивания практических работ

- работы командные, но защита индивидуализирована: каждому участнику задается вопрос по его части и один вопрос по работе в целом;
- если участник не может объяснить представленный материал, по соответствующему критерию баллы не начисляются — это фиксируется в ведомости;
- при повторной сдаче доработанной работы максимальный балл ограничивается 80 % от исходного, если первичная сдача была неудовлетворительной по содержанию.

## Проверка самостоятельности и применения генеративного ИИ

Отдельный «детектор ИИ» не применяется — он ненадежен. Механизм проверки — **устная защита**: вопросы на обоснование выбора, на альтернативы («почему не сделали иначе»), на последствия изменения допущений. Признаки, требующие дополнительных вопросов: несуществующие или неточные ссылки на стандарты и литературу, расчеты без указания допущений, обобщенные формулировки без привязки к предметной области проекта, несогласованность между разделами отчета.

Отсутствие раздела «Использование ИИ-инструментов» при явном применении генеративных моделей — штраф, предусмотренный рубрикой соответствующего КИМ.

## Кейсы индустриальных партнеров

До выдачи темы проверьте: имеется ли согласие правообладателя, каков разрешенный объем использования, доступны ли данные фактически к началу занятия, требуется ли обезличивание. Если данные недоступны — выдайте равнозначную альтернативную тему, чтобы условия аттестации оставались равными для всех обучающихся. Материалы партнеров не размещаются в публичном репозитории дисциплины.

## Проведение новых форм контроля

**Взаимная экспертиза (КИМ-1.3).** Распределяйте работы жеребьевкой и следите, чтобы рецензия была предметной: формулировки «в целом хорошо» не засчитываются. Рецензия, не содержащая ни одного замечания, оценивается не выше 2 баллов — задача экспертизы состоит в выявлении дефектов. Копию рецензии передавайте авторам: учет полученных замечаний оценивается затем при защите проекта.

**Деловая игра (КИМ-2.5).** Раздайте [карточки ролей](../../M2-planning-economics/attachments/business-game-roles.md) обучающимся из других команд заранее. Вводную выбирайте с учетом слабого места конкретного проекта: если команда переоценила точность модели — вводите снижение точности, если недооценила правовые ограничения — закрытие доступа к данным. Обоснованное предложение остановить проект засчитывается как полноценный ответ.

## Использование программных инструментов

Проверку расчетов обучающихся удобно вести эталонным калькулятором:

```bash
python tools/ai_project_economics.py --volume 120000 --manual-cost 55 --accuracy 0.92 \
    --automation-rate 0.7 --capex 3500000 --opex-year 900000 --error-cost 180
```

Варианты контрольной работы формируются генератором; значение `--seed` фиксируйте в ведомости, чтобы восстановить условия при разборе и апелляции:

```bash
python tools/generate_test_variants.py --count 12 --seed 2026 --with-answers --out variants.md
```

Файл с эталонными ответами не публикуется: он внесен в `.gitignore`.

Перед публикацией изменений запускайте проверку репозитория:

```bash
python tools/validate_repository.py --strict
```

## Апелляция и спорные ситуации

Обучающийся вправе запросить разъяснение выставленных баллов в течение трех рабочих дней. Разъяснение дается по критериям рубрики. Пересмотр оценки возможен при выявлении арифметической ошибки или неучтенной части работы; порядок апелляции определяется локальными нормативными актами образовательной организации.

## Учет особых образовательных потребностей

Для обучающихся с ограниченными возможностями здоровья и инвалидов форма проведения контроля устанавливается с учетом индивидуальных психофизических особенностей: при необходимости увеличивается время подготовки, предусматривается доступная форма предъявления заданий и ответов, процедура может проводиться в несколько этапов. Требования к результату при этом не снижаются.
