# Программные инструменты репозитория

Каталог содержит рабочие скрипты, обеспечивающие подготовку и проверку фонда оценочных средств. Все скрипты написаны на Python 3.10+ и **не требуют сторонних библиотек**: их можно запустить на любой машине с установленным Python.

| Скрипт | Назначение | Кто использует |
|---|---|---|
| [`validate_repository.py`](validate_repository.py) | Проверяет непротиворечивость репозитория: структуру, ссылки, покрытие индикаторов, согласованность баллов рубрик с КИМ, наличие незаполненных маркеров | Преподаватель, рецензент |
| [`ai_project_economics.py`](ai_project_economics.py) | Рассчитывает технико-экономические показатели ИИ-решения с учетом доли автоматизации, стоимости ошибок и периодических затрат; строит анализ чувствительности | Преподаватель (эталон проверки), обучающиеся (КИМ-2.2) |
| [`generate_test_variants.py`](generate_test_variants.py) | Формирует индивидуальные варианты контрольной работы (КИМ-2.4) с эталонными ответами; воспроизводим по значению `--seed` | Преподаватель |
| [`make_synthetic_dataset.py`](make_synthetic_dataset.py) | Генерирует синтетический учебный набор данных с заложенными дефектами качества | Преподаватель |

## Быстрый старт

```bash
# проверить репозиторий перед публикацией
python tools/validate_repository.py --strict

# эталонный расчет экономики ИИ-проекта и чувствительности
python tools/ai_project_economics.py --demo --sensitivity accuracy

# сформировать 12 вариантов контрольной работы с ответами
python tools/generate_test_variants.py --count 12 --seed 2026 --with-answers --out variants.md

# пересобрать учебный набор данных
python tools/make_synthetic_dataset.py --rows 500 --seed 2026
```

## Замечание о воспроизводимости

Генераторы вариантов и данных детерминированы: одно и то же значение `--seed` всегда дает один и тот же результат. Это позволяет восстановить условия задания при разборе работ и апелляции, не храня сгенерированные файлы в репозитории.

## Замечание об экземплярах преподавателя

Файлы, сформированные с ключом `--with-answers`, содержат эталонные ответы и **не подлежат публикации** в открытой части репозитория. В `.gitignore` добавлено исключение для файла `variants.md`.
