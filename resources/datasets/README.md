# Наборы данных

Учебные наборы данных для практических работ и проекта. Все наборы **синтетические**: они сгенерированы программно, не содержат персональных данных и не связаны с реальными организациями, поэтому свободно используются и публикуются.

## 1. `support-tickets-sample.csv` — обращения в службу поддержки

Учебный набор для сквозного ИИ-проекта: постановка задачи классификации обращений, оценка пригодности данных, расчет экономики решения.

| Характеристика | Значение |
|---|---|
| Объем | 500 записей |
| Формат | CSV, разделитель `;`, кодировка UTF-8 |
| Период | январь — декабрь 2025 (модельный) |
| Как получен | [`tools/make_synthetic_dataset.py`](../../tools/make_synthetic_dataset.py), `--seed 2026` |
| Персональные данные | отсутствуют |
| Лицензия | CC BY 4.0 (вместе с репозиторием) |

**Поля:** `ticket_id`, `created_at`, `channel`, `text`, `category`, `handling_minutes`, `resolved`.

**Заложенные дефекты.** Набор намеренно несовершенен: обучающиеся должны обнаружить эти особенности при анализе пригодности данных и учесть их в требованиях к системе ИИ и в реестре рисков.

| Дефект | Проявление | Что должен заметить обучающийся |
|---|---|---|
| Дисбаланс классов | Категория «оплата» — около трети записей, «прочее» — менее 7 % | Влияние на метрику качества: доля правильных ответов вводит в заблуждение |
| Пропуски разметки | Около 7 % обращений без категории | Часть данных непригодна для обучения с учителем без доразметки |
| Низкая вариативность текста | 24 уникальные формулировки на 500 записей | Модель, обученная на таких данных, не обобщится на реальные обращения |
| Дубликаты | Повторяющиеся тексты обращений | Риск утечки данных между обучающей и тестовой выборками |
| Выбросы | Отдельные значения времени обработки в 10–20 раз выше медианы | Необходимость робастных оценок при расчете экономии |
| Сдвиг распределения | Во втором полугодии появляется категория «подписка» | Деградация модели во времени; необходимость мониторинга и переобучения |

**Воспроизведение и изменение:**

```bash
python tools/make_synthetic_dataset.py --rows 500 --seed 2026
python tools/make_synthetic_dataset.py --rows 2000 --seed 7 --out my-dataset.csv
```

## 2. `ai-projects-parameters.csv` — параметры условных ИИ-проектов

Набор для сравнительных технико-экономических расчетов: обучающиеся сопоставляют проекты по окупаемости и обосновывают приоритизацию.

| Характеристика | Значение |
|---|---|
| Объем | 12 проектов из различных отраслей |
| Формат | CSV, разделитель `;`, кодировка UTF-8 |
| Персональные данные | отсутствуют |

**Поля:** идентификатор и название проекта, отрасль, годовой объем операций, стоимость ручной обработки, точность модели, доля автоматизации, стоимость ошибки, единовременные и периодические затраты, доступность данных, чувствительность к регулированию.

Набор используется совместно с калькулятором [`tools/ai_project_economics.py`](../../tools/ai_project_economics.py): по каждой строке рассчитываются чистая экономия, срок окупаемости и NPV, после чего проекты ранжируются.

## 3. Правила работы с данными

1. **Приоритет — синтетические и открытые наборы** с ясной лицензией, допускающей учебное использование.
2. Для каждого набора фиксируются: источник, лицензия, наличие персональных данных, порядок обезличивания.
3. **Данные индустриальных партнеров в репозиторий не загружаются.** Допустимо разместить только обезличенное описание структуры данных и ссылку на условия доступа.
4. Персональные данные используются исключительно в обезличенном виде (Федеральный закон от 27.07.2006 № 152-ФЗ).
5. Сведения, составляющие коммерческую тайну, обрабатываются в объеме, разрешенном соглашением о конфиденциальности, и не публикуются.
6. Если доступ к данным партнера отсутствует или ограничен, используются наборы из настоящего раздела либо равнозначная альтернативная тема проекта. Условия аттестации при этом остаются равными для всех обучающихся.

## 4. Крупные файлы

Наборы объемом более 50 МБ в репозиторий не помещаются: размещается карточка набора со ссылкой на внешнее хранилище и параметрами воспроизведения.
