# Промпты для работы с генеративными моделями

Раздел содержит проверенные промпты, применяемые преподавателем при подготовке материалов и обучающимися — в разрешенных пределах (см. [правила](../../methodical-guidelines/students/README.md)).

## Для преподавателя

| Задача | Промпт (шаблон) | Примечание |
|---|---|---|
| Генерация вариантов учебного кейса | «Сформулируй описание бизнес-процесса в отрасли [отрасль] объемом до 200 слов, содержащее проблему, которую целесообразно решать методами машинного обучения. Не указывай решение» | Проверить реалистичность и отсутствие отсылок к реальным организациям |
| Генерация вариантов задачи на окупаемость | «Составь условие задачи: объем операций [N] в год, стоимость ручной обработки [X], точность модели [P]. Требуется рассчитать экономию и срок окупаемости» | Проверить арифметическую согласованность |

## Для обучающихся

| Задача | Промпт (шаблон) | Ограничение |
|---|---|---|
| Проверка формулировки требования | «Оцени, является ли требование проверяемым, и предложи, как сделать его измеримым: [текст требования]» | Итоговую формулировку команда принимает самостоятельно |
| Структурирование отчета | «Предложи структуру отчета по разделам: [перечень]» | Содержание разделов пишется командой |

> **Запрещено** включать в промпты данные индустриальных партнеров, персональные данные и сведения, составляющие коммерческую тайну. Любой результат генерации проверяется по первоисточникам: сгенерированные ссылки на стандарты, публикации и числовые данные нередко оказываются недостоверными.
