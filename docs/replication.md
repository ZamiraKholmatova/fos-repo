# Тиражирование материалов на другие дисциплины

Фонд оценочных средств спроектирован так, чтобы его можно было перенести на смежные дисциплины в области ИИ с ограниченными трудозатратами. Документ объясняет, что переносится без изменений, что требует адаптации и в каком порядке действовать.

## 1. Что переносится без изменений

| Элемент | Файлы | Почему переносим |
|---|---|---|
| Структура репозитория | все каталоги | Не зависит от предметного содержания |
| Модель измерения (форма таблицы) | `README.md`, раздел 2 | Форма «компетенция → индикатор → ступень → дескриптор → форма контроля → КИМ» универсальна для любой дисциплины на базе КРМ |
| Система ступеней Б₀–Б₃ | `docs/levels-trajectory.md` | Ступени описывают тип познавательного действия, а не содержание |
| Структура рубрики | все `rubric-*.md` | Схема «критерий → индикатор → 0 / половина / максимум» не зависит от темы |
| Правила по генеративному ИИ | все КИМ, методические указания | Единые для любой дисциплины |
| Правила работы с кейсами партнеров | `Project/README.md`, `resources/datasets/README.md` | Определяются законодательством, а не темой курса |
| Валидатор репозитория | `tools/validate_repository.py` | Проверяет формальную непротиворечивость, не привязан к содержанию |

## 2. Что требует адаптации

| Элемент | Объем работы | Что менять |
|---|---|---|
| Коды компетенций и индикаторов | 1–2 часа | Выбрать из КРМ индикаторы своей дисциплины, перенести дескрипторы |
| Дескрипторы уровней | 2–3 часа | Переписать под предметное действие, сохранив различение ступеней |
| Содержание заданий КИМ | 4–8 часов | Заменить предметную задачу, сохранив тип КИМ и структуру рубрики |
| Наборы данных | 2–4 часа | Сгенерировать свои через `tools/make_synthetic_dataset.py` либо подобрать открытые |
| Список ресурсов | 1–2 часа | Заменить литературу, стандарты и инструменты |

## 3. Типовые КИМ и их переносимость

| Тип КИМ | Пример в репозитории | На какие дисциплины переносится |
|---|---|---|
| Аналитическая работа с постановкой задачи | КИМ-1.1 | Любая дисциплина, где есть переход от предметной проблемы к формальной постановке |
| Разработка документа по стандарту | КИМ-1.2 | Дисциплины по требованиям, архитектуре, документированию систем |
| Взаимная экспертиза (peer review) | КИМ-1.3 | Любая дисциплина с проектными артефактами; особенно эффективна там, где важна культура ревью |
| Расчетно-инструментальная работа | КИМ-2.1, КИМ-2.2 | Дисциплины с количественной оценкой: экономика ИИ, оценка качества моделей, планирование |
| Организационно-командная работа | КИМ-2.3 | Дисциплины с командным форматом обучения |
| Контрольная с генерируемыми вариантами | КИМ-2.4 | Любая дисциплина с расчетной частью; генератор адаптируется заменой пула задач |
| Деловая игра с ролями | КИМ-2.5 | Дисциплины, где требуется защита решения перед стейкхолдерами |
| Интегральный проект | `Project/` | Любая проектно-ориентированная дисциплина |

## 4. Порядок переноса

1. Скопируйте репозиторий и замените наименования модулей.
2. Определите индикаторы КРМ своей дисциплины и заполните модель измерения.
3. Постройте траекторию ступеней по образцу `docs/levels-trajectory.md`: решите, какие индикаторы доводятся до Б₃, а какие остаются на Б₁.
4. Для каждого индикатора выберите тип КИМ из таблицы выше и адаптируйте задание.
5. Пересоберите баллы так, чтобы сумма составляла 100, и приведите рубрики в соответствие.
6. Запустите `python tools/validate_repository.py --strict` и устраните ошибки.

## 5. Оценка трудозатрат

Полный перенос на смежную дисциплину при сохранении структуры — ориентировочно **20–30 часов** работы преподавателя. Разработка аналогичного фонда с нуля, по опыту подготовки настоящего репозитория, требует существенно больше: только согласование модели измерения, рубрик и системы баллов занимает около половины этого времени.
