# Фонд оценочных средств по дисциплине «Проектная деятельность»

Репозиторий содержит фонд оценочных средств (ФОС) по дисциплине **«Проектная деятельность»** — курсу об управлении жизненным циклом систем искусственного интеллекта: от анализа бизнес-проблемы и требований к системе ИИ до планирования, оценки экономических эффектов и защиты проектного решения перед стейкхолдерами.

Результаты обучения связаны с компетентностно-ролевой моделью в области искусственного интеллекта (**КРМ версии 3.0**), блок «Жизненный цикл систем искусственного интеллекта».

| Параметр | Значение |
|---|---|
| Образовательная программа | 09.03.01 «Информатика и вычислительная техника», профиль «ТОП ИИ» |
| Уровень образования | Бакалавриат |
| Объем | 108 а.ч. / 3 з.е. |
| Формат обучения | Очный |
| Промежуточная аттестация | Зачет с оценкой |
| Целевые роли КРМ | **AI PM** (менеджер проектов ИИ), **AI CDO** (руководитель цифровой трансформации) |
| Формируемые компетенции | LC-1, LC-4, LC-6, LC-7 (11 индикаторов) |
| Виды КИМ | 8 контрольно-измерительных материалов шести различных типов |

## 1. О дисциплине

**Цель.** Сформировать способность организовывать и вести проектную деятельность по созданию систем искусственного интеллекта на всех этапах их жизненного цикла — от анализа бизнес-проблемы и оценки перспективности применения ИИ до планирования, реализации, контроля и оценки эффективности ИИ-проекта в составе команды.

**Чем дисциплина отличается от типового курса управления проектами.** Предметом выступает не проект вообще, а **жизненный цикл ML-системы** со свойственными ему особенностями: неопределенность достижимого качества модели на старте, зависимость результата от доступности и качества данных, деградация модели после внедрения, стоимость ошибки как экономический параметр, правовой режим данных. Эти особенности заложены в задания: обучающиеся обосновывают выбор между CRISP-DM и CRISP-ML(Q), рассчитывают экономику решения с учетом доли автоматизации и стоимости ошибок, составляют реестр рисков, специфичных для систем ИИ, и защищают план перед комитетом, в котором присутствует специалист по правовому режиму данных.

**Структура.** Два содержательных модуля, сквозной командный ИИ-проект и промежуточная аттестация. Лекции — 14 а.ч., практические занятия — 36 а.ч., самостоятельная работа — 56 а.ч.

**Планируемые результаты.** По завершении дисциплины обучающийся сможет:

1. формализовать проблему предметной области и обосновать перспективность применения ИИ для ее решения;
2. разработать устав ИИ-проекта и фрагмент технического задания на систему ИИ;
3. построить план ИИ-проекта (ИСР, календарный план, бюджет) и обосновать выбор методологии разработки;
4. составить реестр рисков ИИ-проекта и план реагирования, оценить технико-экономические показатели решения;
5. распределить роли в команде ИИ-проекта и организовать ее взаимодействие;
6. оценить экономические эффекты внедрения ИИ и предложить элементы стратегии внедрения;
7. провести экспертизу чужого проектного решения и защитить собственное перед стейкхолдерами.

## 2. Модель измерения

Уровни КРМ: **Б** — базовый, **С** — средний, **П** — продвинутый. Внутри дисциплины освоение проходит четыре наблюдаемые ступени: **Б₀** — ознакомление, **Б₁** — воспроизведение по образцу, **Б₂** — самостоятельное применение, **Б₃** — перенос на новую ситуацию (граница выхода на уровень С). Как происходит переход между ступенями, описано в документе [«Траектория освоения компетенций»](docs/levels-trajectory.md).

## 3. Контрольно-измерительные материалы

Восемь КИМ шести различных типов: аналитическая работа, разработка документа по стандарту, взаимная экспертиза, расчетно-инструментальная работа, деловая игра, контрольная работа с генерируемыми вариантами, интегральный проект, устное собеседование.

| КИМ | Тип | Индикаторы | Ступень | Баллы |
|---|---|---|---|---:|
| [1.1. Анализ бизнес-проблемы и оценка перспективности ИИ](M1-ai-project-initiation/kim-01-practical-work.md) | Аналитическая работа (командная) | LC-1.1, LC-6.3 | Б₂ | 10 |
| [1.2. Устав ИИ-проекта и техническое задание](M1-ai-project-initiation/kim-02-practical-work.md) | Разработка документа по стандарту | LC-1.1, LC-1.3 | Б₂ | 12 |
| [1.3. Взаимная экспертиза постановки задачи](M1-ai-project-initiation/kim-03-peer-review.md) | **Взаимная экспертиза (индивидуальная)** | LC-1.1, LC-1.3, LC-6.3 | **Б₃** | 5 |
| [2.1. План проекта: ИСР, календарный план, методология](M2-planning-economics/kim-01-practical-work.md) | Расчетно-инструментальная работа | LC-4.1 | Б₂ | 8 |
| [2.2. Ресурсы, бюджет и риски ИИ-проекта](M2-planning-economics/kim-02-practical-work.md) | Расчетно-инструментальная работа | LC-4.1, LC-7.1, LC-7.3 | Б₂ | 10 |
| [2.3. Команда ИИ-проекта: роли, RACI, мониторинг](M2-planning-economics/kim-03-practical-work.md) | Организационно-командная работа | LC-4.1, LC-4.2 | Б₂ | 5 |
| [2.4. Контрольная работа](M2-planning-economics/kim-04-test.md) | Рубежный контроль (генерируемые варианты) | LC-4.1, LC-7.1, LC-7.3 | Б₁–Б₂ | 5 |
| [2.5. Деловая игра «Проектный комитет»](M2-planning-economics/kim-05-business-game.md) | **Ролевая симуляция** | LC-4.1, LC-4.2, LC-7.1 | **Б₃** | 5 |
| [Защита итогового ИИ-проекта](Project/README.md) | Интегральный проект (командный) | LC-6.1–6.4, LC-7.2, LC-4.2 | Б₁–Б₂ | 25 |
| [Зачет с оценкой](Exam/README.md) | Устное собеседование | все индикаторы | подтверждение | 15 |

Каждый КИМ содержит назначение, проверяемые результаты с указанием ступени, условия выполнения, материалы задания, формат сдачи, критерии и шкалу оценивания, правила использования внешних ресурсов и генеративного ИИ. К заданиям приложены рабочие шаблоны документов (устав проекта, требования, реестр рисков, матрица RACI, карточки ролей, форма рецензии).

## 5. Программные инструменты и данные

Репозиторий содержит рабочие скрипты (Python 3.10+, без сторонних зависимостей) и учебные наборы данных.

| Инструмент | Назначение |
|---|---|
| [`validate_repository.py`](tools/validate_repository.py) | Автоматическая проверка репозитория: структура, ссылки, покрытие индикаторов, согласованность баллов рубрик с КИМ |
| [`ai_project_economics.py`](tools/ai_project_economics.py) | Расчет технико-экономических показателей ИИ-решения с анализом чувствительности; эталон проверки КИМ-2.2 |
| [`generate_test_variants.py`](tools/generate_test_variants.py) | Формирование воспроизводимых вариантов контрольной работы с эталонными ответами |
| [`make_synthetic_dataset.py`](tools/make_synthetic_dataset.py) | Генерация синтетического учебного набора данных с заложенными дефектами качества |

| Набор данных | Назначение |
|---|---|
| [`support-tickets-sample.csv`](resources/datasets/support-tickets-sample.csv) | 500 синтетических обращений с намеренными дефектами: дисбаланс классов, пропуски разметки, дубликаты, выбросы, сдвиг распределения |
| [`ai-projects-parameters.csv`](resources/datasets/ai-projects-parameters.csv) | Параметры 12 условных ИИ-проектов для сравнительных технико-экономических расчетов |

Данные полностью синтетические: они не содержат персональных данных и не связаны с реальными организациями, поэтому свободно используются и публикуются. Подробнее — в [карточке ресурсов](resources/datasets/README.md).

## 6. Тиражирование

Структура репозитория, система ступеней освоения, форма рубрик, правила по генеративному ИИ и валидатор переносятся на другие дисциплины области ИИ без изменений; адаптации требуют предметное содержание заданий и перечень индикаторов. Порядок переноса и оценка трудозатрат — в документе [«Тиражирование материалов»](docs/replication.md).
